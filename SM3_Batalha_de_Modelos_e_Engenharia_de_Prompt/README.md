# Batalha de Modelos & Engenharia de Prompt (XML) 🚀

Este projeto apresenta um estudo comparativo de diferentes modelos de Inteligência Artificial (LLMs) submetidos a um prompt estruturado em formato XML para a criação de uma página web.

## 📋 Sobre o Desafio
O objetivo principal foi construir um **Prompt Estruturado em XML** capaz de gerar uma página HTML5 Single Page com CSS3 integrado.

### Estrutura do Prompt Utilizada
O prompt seguiu um protocolo rigoroso de execução contendo:
**Objetivo:** Criar página HTML5 única com CSS3 interno.
**Diretrizes de Design:** Layout responsivo, minimalista e definições de tipografia.
**Obrigatoriedades Técnicas:** Menu de navegação (âncoras), galeria/portfólio e rodapé de contato.
**Métrica de Saída:** Estimativa de tokens gerados ao final do código.

## 📊 Análise Comparativa
Os testes foram realizados nos modelos: **ChatGPT, Gemini, DeepSeek, Qwen, Grok, Maritaca e Claude**.

### Performance por Modelo
| Modelo | Precisão do HTML | Criatividade | Consumo de Tokens |
| :--- | :--- | :--- | :--- |
| **Gemini** | Alta  (Atendeu expectativas) | Muito bom/Visual bonito | 264 |
| **Claude** | Alta (Estruturado/Moderno) | Uso de imagens de destaque | 5.200 |
| **GPT** | 80% de precisão | Simples, sem imagens | 1.100 |
| **DeepSeek** | Apenas código (sem visualização) | Baixa criatividade | 2.150 |
| **Maritaca** | Erro de cores | Nenhuma (sem visualização) | 1.100 |

## 🧠 Reflexão Crítica
**Maior Compreensão de XML:** O modelo **Gemini** demonstrou a melhor interpretação da estrutura proposta.
**Verbosidade vs. Qualidade:** Modelos com maior número de tokens nem sempre entregaram o melhor resultado; os tokens extras geralmente refletiam mais conteúdo, não necessariamente melhor qualidade.
**Veredito para Prototipagem:** * **Prototipagem Rápida:** Gemini.
